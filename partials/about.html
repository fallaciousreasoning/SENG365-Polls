Overview

A simple MVVM app for viewing polls. This app was made by Jay Harris as part of SENG365 in 2015.

The aim of this application is to be a client side app for viewing polls which requests the data it needs via rest API calls. The rest API supports the following methods.

Features

  • View Polls
  • Vote on Polls and View Votes (note: This link won't take you anywhere if you don't have any polls)
  • Administer Polls From here you can add, remove and edit polls
  • View an Awesome About Page (if you're seeing this you probably don't need the link...)

Styling

This website uses Bootstrap for almost all of it's css (there is a very limited amount that has been added by me, mostly for indenting and weighting text).

Known Bugs

If you have a slow internet connection it is sometimes possible to vote multiple times in a poll (as the client waits until it has relieved the list of votes from the server before disabling voting).

Another problem I ran into was that I couldn't access put data in code igniter. As a temporary fix for this I have wrapped the put method in the post, so posting to /services/polls/{pollId} will update an existing poll.

You can also save a poll with blank fields (such as the title, question or any of the answers). On a related note you can also save polls with no answers...). What I'm trying to say is I don't really check if your polls are valid :/

REST API

The REST services provided by this program can be accessed by {BASEURL}/services/{query}. In all cases the program will respond with a 404 if something goes wrong or a 20X if all is right in the world. Now, to the details.
Service URL Method Request Data Response Data Explanation
{BASEURL}/index.php/service/polls GET None 200 OK and a JSON list of polls. Each poll is a JSON object with an id, title, question and a list of answers Gets a list of all available polls
{BASEURL}/index.php/service/polls/{pollId} GET None 200 OK and a JSON object representing the poll with the specified id. If the id doesn't represent a valid poll a 404 will be returned. Gets a specific poll
{BASEURL}/index.php/service/polls POST A JSON object representing the poll to create 201 if the poll was successfully created or a 404 if something went wrong Creates a new poll
{BASEURL}/index.php/service/polls/{pollId} POST A JSON object representing the poll to update 200 if the poll was successfully updated or a 404 if something went wrong Updates an existing poll. Note, in the original specification this method was meant to be PUT, however, the CodeIgniter library I was using to make a rest service had significant problems with PUT, so I have made it POST as a workaround
{BASEURL}/index.php/service/polls/{pollId} DELETE None 200 if the poll was successfully deleted Deletes an existing poll
{BASEURL}/index.php/service/votes/{pollId}/{optionNo} POST None None Votes for a an option within a poll. {pollId} is the id of the poll to vote for and {optionNo} is the option within the poll to vote for
{BASEURL}/index.php/service/votes/{pollId} GET None 200 ok and a list of all votes on a poll of the form {0, 1, 5} where the index of the vote count is the same as that as the optionNumber of an answer within the poll. 404 if the pollId is invalid. Gets a list of all votes within a poll. {pollId} is the id of the poll to get votes for
{BASEURL}/index.php/service/votes/{pollId} DELETE None 200 ok. Resets the votes on a poll. {pollId} is the id of the poll to reset
